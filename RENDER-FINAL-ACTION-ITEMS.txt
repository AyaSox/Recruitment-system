CURRENT ISSUE:
Dockerfile has encoding corruption preventing Docker build on Render

SOLUTION:
The Dockerfile needs to be recreated locally with proper UTF-8 encoding

EXACT STEPS TO FIX:

1. On your local machine, open your IDE (VS Code or Visual Studio)

2. In the file explorer, navigate to the root directory

3. DELETE the corrupted Dockerfile

4. CREATE a new file called "Dockerfile" (no extension)

5. PASTE this exact content:

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY ATSRecruitSys.Server/ATSRecruitSys.Server.csproj ATSRecruitSys.Server/
WORKDIR /src/ATSRecruitSys.Server
RUN dotnet restore ATSRecruitSys.Server.csproj
COPY ATSRecruitSys.Server/ .
RUN dotnet build ATSRecruitSys.Server.csproj -c Release -o /app/build

FROM build AS publish
RUN dotnet publish ATSRecruitSys.Server.csproj -c Release -o /app/publish /p:UseAppHost=false

FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS final
WORKDIR /app
COPY --from=publish /app/publish .
EXPOSE 10000
ENTRYPOINT ["dotnet", "ATSRecruitSys.Server.dll"]

6. SAVE the file

7. COMMIT and PUSH to GitHub:
   git add Dockerfile
   git commit -m "Fix Dockerfile encoding for Render deployment"
   git push origin main

8. ON RENDER:
   - Go to backend service
   - Click Manual Deploy
   - Select Deploy latest commit
   - Wait 5-7 minutes
   - Should build successfully now

THEN:

9. ADD ENVIRONMENT VARIABLES:
   In backend service, add these 7 variables:
   
   ASPNETCORE_ENVIRONMENT = Production
   ASPNETCORE_URLS = http://+:10000
