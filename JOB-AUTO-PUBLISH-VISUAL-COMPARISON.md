## Real User Scenarios

### Scenario 1: Recruiter Posts Job
**BEFORE ?**
```
"I created the job but it says Draft. 
Do I need to do something else? 
Oh, there's a Publish button. 
Why didn't it publish automatically?"
```

**AFTER ?**
```
"Great! Job is live. 
Candidates can now apply."
```

---

### Scenario 2: Admin Posts Urgent Job
**BEFORE ?**
```
"Urgent job needs to be live NOW!
Created it... wait, it's in Draft?
Where's the publish button?
Found it, clicking...
Finally live. That took too long."
```

**AFTER ?**
```
"Urgent job needs to be live NOW!
Created it... done! 
It's live immediately. Perfect!"
```

---

### Scenario 3: Manager Wants Draft First
**BEFORE ?**
```
"Created as draft. Good.
Now I can review it.
Ready to publish? Click publish."
```

**AFTER ?**
```
"Created... it's live already.
Let me unpublish it quickly.
Now it's draft. I can review.
Ready to publish? Click publish."
```

**Note**: Slightly more steps for this rare case, 
but vast majority of users want instant publishing.
